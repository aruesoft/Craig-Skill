# 학습 파이프라인 설계안 (Telegram → Obsidian → Retro)

> 작성일: 2026-07-05 · 상태: Draft v1
> 목표: 흩어진 학습 자료를 텔레그램/옵시디언으로 수집하고, PARA 볼트에 지식으로 통합하며, 간격 반복으로 장기 기억화하는 파이프라인

---

## 1. 전체 아키텍처

```
[입력]                          [처리: Claude Code 스킬]              [출력]

텔레그램 봇 ──┐
 (텍스트/링크/  │   ①ingest        ②curate           ③retro
  유튜브/쓰레드/ ├─→ 00_Inbox ──→ 가치 평가 ──→ Areas/Resources ──→ 복습 큐
  이미지/노트캡쳐)│   (Raw 보존)    (승격/병합/보관)   (주제노트+MOC)      │
옵시디언 직접 ──┘                                                      ▼
 입력                                                        텔레그램 복습 알림
                                                             주간 리트로 리포트
```

**프로젝트의 핵심 가치**

1. **Inbox = 수집**: 정보·학습 링크를 마찰 없이 모은다 (Raw 보존)
2. **학습 노트 = 판단을 거친 지식**: Claude가 큐레이팅을 제안하되, **Craig의 승인을 거친 것만** 주제 노트로 승격·정리된다
3. **복습과 연결이 목적**: 승격된 지식은 간격 반복으로 장기 기억화하고(retro), 노트 간 링크·태그로 지식 그래프를 만든다(garden)

**설계 원칙**

- **봇과 지능의 분리**: 텔레그램 봇(기존 모듈 + launchd 상시 실행)은 "수신 → 파일 저장 → 발신"만 담당. 요약·분류·통합·복습 생성 등 지능 처리는 전부 Claude Code 스킬이 담당. 봇은 단순해서 안 죽고, 지능은 스킬 수정만으로 개선 가능.
- **원본 불변**: Inbox의 Raw 원본은 절대 수정하지 않음. 가공은 항상 별도 섹션/노트로.
- **핸드오프는 파일로**: 봇 ↔ Claude 간 통신은 JSON 큐 파일. 프로세스 간 결합 없음.

## 2. 옵시디언 볼트 구조 (PARA)

기존 볼트에 아래 구조를 추가한다.

```
Vault/
├── 00_Inbox/                  # ① Raw Data 착지점
│   ├── 2026-07-05_llm-agent-memory.md
│   └── _attachments/          # 이미지·캡쳐 원본
├── 01_Projects/               # 마감 있는 활성 작업 (기존 유지)
├── 02_Areas/                  # 지속 학습 영역
│   ├── AI-ML/
│   │   ├── _MOC_AI-ML.md      # 영역 지도 (Map of Content)
│   │   ├── LLM 에이전트 설계.md  # 주제 노트 (지식이 쌓이는 곳)
│   │   └── ...
│   └── Business-Investing/
│       ├── _MOC_Business-Investing.md
│       └── ...
├── 03_Resources/              # Areas에 안 붙는 참고자료
├── 04_Archive/                # 처리 완료된 Inbox 원본, 가치 낮은 자료
└── _System/
    ├── Templates/             # inbox / topic / retro 템플릿
    ├── Queue/
    │   ├── incoming/          # 봇이 쓰는 수신 JSON
    │   ├── outgoing/          # 스킬이 쓰는 발신 JSON (봇이 소비)
    │   └── processed/         # 처리 완료 이동
    └── Review/
        ├── daily-queue.md     # 오늘의 복습 목록 (옵시디언에서도 확인)
        └── 2026-W27_retro.md  # 주간 리트로 리포트
```

카테고리는 AI-ML, Business-Investing 2개로 시작하고, curate 단계에서 어느 쪽에도 안 붙는 자료가 반복되면 Claude가 새 Area/Resource 신설을 제안한다.

## 3. 노트 스키마

### 3-1. Inbox 노트 (Raw)

```yaml
---
type: inbox
captured: 2026-07-05T09:30
source: telegram          # telegram | manual
source_type: youtube      # youtube | thread | web | image | text | note-capture
source_url: https://...
status: raw               # raw → processed → promoted | archived
suggested_area: AI-ML
value: high               # high | mid | low (curate 때 확정)
promoted_to: "[[LLM 에이전트 설계]]"
---
## 원본
(텍스트 전문 / 링크 본문 추출 / 유튜브 transcript / 이미지 임베드 + OCR 텍스트)

## 요약
(Claude 3~5줄 요약)

## 핵심 포인트
- ...

## 연결 후보
- [[기존 주제노트A]] — 이유
```

### 3-2. 주제 노트 (Category Note — 지식이 통합되는 단위)

```yaml
---
type: topic
area: AI-ML
created: 2026-06-01
updated: 2026-07-05
maturity: growing         # seed → growing → evergreen
review:
  interval: 7             # 일 단위
  next_review: 2026-07-12
  reps: 3
  lapses: 0
sources: 5                # 통합된 자료 수
---
## 핵심 요지 (내 언어로)
## 상세
## 열린 질문
## 출처
- [[2026-07-05_llm-agent-memory]] (inbox 원본 링크)
```

- 새 자료가 들어오면 **새 노트를 만들기보다 기존 주제 노트에 병합**하는 것이 기본. 병합할 노트가 없을 때만 신규 생성.
- MOC 노트는 curate가 자동 갱신 (신규 주제 추가, maturity 표시).

## 4. 파이프라인 단계별 설계

### ① learn-ingest (수집 → 1차 정리)

- **트리거**: 봇이 `Queue/incoming/`에 JSON 저장 후 실행 (또는 30분 배치 / 수동)
- **입력 JSON**: `{ts, chat_id, msg_id, text, urls[], attachments[], tags[]}`
- **처리**:
  1. URL 타입 판별 → 콘텐츠 추출
     - 유튜브/인스타 동영상: **전사 파이프라인**(아래 ①-a) → 스크립트 전문 + 구조화 정리
     - 쓰레드/X: 스레드 텍스트 추출
     - 웹: 본문 추출 (readability)
     - 이미지/노트캡쳐: `_attachments/` 저장 + Claude 비전으로 텍스트 전사·내용 설명
  2. 중복 검사: URL 정규화 해시로 기존 Inbox/Archive 검색 → 중복이면 기존 노트에 재수집 메모만 추가
  3. Inbox 노트 생성 (원본 + 요약 + 핵심 포인트 + 카테고리 제안 + 연결 후보)
  4. 텔레그램 확인 발신: `✅ 저장됨: "제목" → 제안: AI-ML · 연결후보 2건`
- **사용자 힌트**: 메시지에 `#ai` `#biz` 태그를 붙이면 카테고리 제안을 오버라이드

### ①-a 동영상 전사 파이프라인 (유튜브 · 인스타 릴스)

동영상 링크는 "스크립트 전문 + 정리"까지 자동 생성한다.

```
링크 수신
  → [유튜브] youtube-transcript-api로 자막 시도 (자동생성 자막 포함, ko/en)
       ├ 성공 → 자막 텍스트 확보
       └ 실패(자막 없음) ↓
  → [공통 폴백 / 인스타 기본 경로]
     yt-dlp로 오디오만 다운로드 (m4a, _attachments/media/)
       → Whisper STT (로컬 whisper.cpp 또는 OpenAI API, 언어 자동감지)
       → 스크립트 텍스트 확보
  → Claude 정리: 타임스탬프 섹션 나누기 + 요약 + 핵심 포인트
```

- **메타데이터**: yt-dlp로 제목·채널(계정)·길이·업로드일 함께 수집
- **Inbox 노트 구성** (동영상 전용 섹션):

```markdown
## 원본
- 출처: [제목](URL) · 채널명 · 12:34 · 2026-07-01

## 스크립트 (전문)
### [00:00] 도입
...전사 텍스트...
### [03:12] 핵심 주장
...

## 요약
## 핵심 포인트
```

- **길이 제한**: 30분 초과 영상은 텔레그램으로 확인 질문 후 진행 (STT 비용/시간 관리)
- **인스타 주의사항**: 비공개/로그인 필요 게시물은 yt-dlp 쿠키 파일(`--cookies`)로 처리, 그래도 실패하면 `transcript: failed` 마킹 + 링크만 저장 후 텔레그램으로 실패 알림
- **음악/무발화 영상**: STT 결과가 거의 비면 캡션·설명 텍스트로 대체하고 노트에 표시
- **텔레그램 확인 발신에 전사 결과 포함**: `✅ 저장됨: "제목" · 스크립트 2,400자 전사 완료 → 제안: AI-ML`

### ② learn-curate (가치 평가 → 통합)

- **트리거**: 1일 1회 (예: 22:00) + 수동 `/curate`
- **원칙**: **최종 승격 판단은 Craig가 한다.** Claude는 가치 평가·병합 위치까지만 제안하고, 승인된 것만 학습 노트로 들어간다.
- **처리**: `status: raw|processed`인 Inbox 노트 각각에 대해
  1. **가치 제안** — Claude가 high/mid/low 판정 + 근거 한 줄 작성
  2. **승인 요청 (텔레그램)**: 노트별 카드 발송 — 제목 · 요약 2줄 · 제안(`→ [[LLM 에이전트 설계]]에 병합` 등) + 인라인 버튼: ✅ 승인 · 📁 보관만 · 🗑 버림 · ✏️ 다른 위치 (응답은 큐로 수신, 무응답분은 다음 curate 때 재질문)
  3. **병합 우선** (승인 건 처리): 제목·태그·MOC 기반으로 관련 주제 노트 탐색 → 있으면 해당 노트에 새 섹션/포인트로 병합하고 `updated`·`sources` 갱신, 없으면 신규 주제 노트 생성 (`maturity: seed`, 복습 스케줄 시작)
  3. 위키링크 연결 및 MOC 갱신
  4. Inbox 원본은 `status: promoted` 마킹 후 유지 (또는 90일 후 Archive 이동)
  5. 결과 요약을 텔레그램 발신: `📦 오늘 정리: 승격 2 · 병합 3 · 보관 1`

### ②-a learn-garden (볼트 정비 — 링크·태그 자동 관리)

큐레이트된 노트들을 정기적으로 재검토해 지식 그래프를 가꾸는 단계.

- **트리거**: 주 2회 (수·일 21:00) + 수동 `/garden`
- **처리** (`02_Areas`, `03_Resources`의 topic 노트 전체 대상):
  1. **링크 발굴**: 노트 본문에서 다른 주제 노트의 제목·별칭이 평문으로 언급되면 `[[위키링크]]`로 변환. 직접 언급이 없어도 내용상 관련이 강한 노트 쌍은 `## 관련 노트` 섹션에 상호 링크 + 한 줄 이유 추가
  2. **태그 정비**:
     - 노트 내용 기반으로 frontmatter `tags` 자동 부여/갱신 (예: `#llm`, `#agent`, `#valuation`)
     - 유사 태그 통합(`#머신러닝` → `#ml`), 고아 태그 제거 — 태그 사전은 `_System/tags.md`에서 관리하고 신규 태그는 여기에 자동 등록
  3. **고아 노트 구제**: 들어오는 링크가 0인 노트를 찾아 MOC 또는 관련 노트에 연결
  4. **MOC 리프레시**: 영역별 노트 수·maturity 분포 갱신, 노트가 충분히 모인 하위 주제는 서브 MOC 분리 제안
  5. 변경 내역을 `_System/Review/garden-log.md`에 기록하고 텔레그램 요약 발신: `🌱 정비: 링크 +7 · 태그 12건 갱신 · 고아 노트 2건 연결`
- **안전 원칙**: 본문 의미는 건드리지 않음 — 링크 변환·태그·관련노트 섹션·MOC만 수정. 대량 변경(20건 초과)은 실행 전 텔레그램으로 승인 요청

### ③ learn-retro (일일 복습)

- **트리거**: 매일 08:00
- **간격 반복 (SM-2 라이트)**: `1 → 3 → 7 → 21 → 60일`
  - 응답 버튼: 👍 쉬움(다음 단계 ×1.3) · 👌 보통(다음 단계) · 👎 어려움(1일로 리셋, lapses+1)
- **처리**:
  1. `next_review ≤ 오늘`인 주제 노트 추출 (일일 상한 5개, 초과분은 이월)
  2. 노트별 **복습 카드** 생성: 3줄 요약 + 능동 회상 질문 1~2개 (답은 접어서/다음 메시지로)
  3. 텔레그램 발송, 응답 수신 시 봇이 `Queue/incoming/`에 응답 JSON 저장 → 다음 ingest/retro 실행 때 frontmatter의 interval 갱신
  4. `_System/Review/daily-queue.md`에도 동일 목록 기록 (옵시디언에서 복습하는 날 대비)

### ④ learn-weekly (주간 리트로 리포트)

- **트리거**: 일요일 20:00
- **내용**: 이번 주 수집/승격/병합 통계, 새로 생긴 연결(링크 그래프 변화), 복습 성과(쉬움/어려움 비율), **약한 주제** (lapses 많거나 열린 질문 미해결), 다음 주 복습 예정 목록, Claude의 한 줄 코멘트(패턴 관찰)
- **출력**: `_System/Review/2026-W27_retro.md` 생성 + 텔레그램으로 요약 발송

## 5. 텔레그램 봇 인터페이스

기존 봇 모듈(launchd 상시 실행)에 추가할 최소 기능:

| 방향 | 기능 |
|---|---|
| 수신 | 모든 메시지(텍스트/링크/사진/파일) → `Queue/incoming/` JSON + 첨부 저장 |
| 수신 | 복습 카드 인라인 버튼 콜백(👍👌👎) → 응답 JSON 저장 |
| 발신 | `Queue/outgoing/` 폴링 → sendMessage/sendPhoto 후 `processed/` 이동 |

명령어:

- (기본) 아무 메시지나 전송 = 수집
- `/status` — Inbox 대기 수, 오늘 복습 잔여, 주간 통계
- `/review` — 즉시 복습 카드 1장
- `/find 키워드` — 볼트 검색 후 상위 결과 링크(옵시디언 URI) 반환
- `/curate` — 수동 정리 트리거
- `/garden` — 수동 링크·태그 정비 트리거
- `#태그` — 카테고리 힌트

## 6. 실행/스케줄 구성 (Claude Code)

| 작업 | 주기 | 방식 |
|---|---|---|
| bot (수신/발신) | 상시 | 기존 모듈 + launchd |
| learn-ingest | 수신 트리거 또는 30분 | launchd → `claude -p` 헤드리스 실행 |
| learn-curate | 매일 22:00 | launchd 스케줄 |
| learn-garden | 수·일 21:00 | launchd 스케줄 |
| learn-retro | 매일 08:00 | launchd 스케줄 |
| learn-weekly | 일 20:00 | launchd 스케줄 |

스킬 구성: `.claude/skills/` 아래 `learn-ingest`, `learn-curate`, `learn-garden`, `learn-retro`, `learn-weekly` 5개. 공통 설정은 `pipeline.config.json` 하나로 (볼트 경로, 큐 경로, 카테고리 목록, SRS 파라미터, 일일 복습 상한).

## 7. 엣지 케이스 & 운영 원칙

- **전사 실패**: 자막 API → yt-dlp+Whisper 순서로 폴백, 최종 실패 시 제목·설명만으로 요약하고 `transcript: failed` 표시 → 나중에 수동 보완 가능
- **STT 의존성**: yt-dlp + ffmpeg + whisper.cpp(로컬, 무료) 또는 OpenAI Whisper API(고품질, 유료) — config에서 선택
- **중복 수집**: URL 정규화(쿼리 파라미터 제거) 해시로 dedupe
- **옵시디언 동기화 충돌**: 스킬은 노트를 열어두는 시간 최소화, frontmatter 갱신은 원자적 쓰기
- **분류 애매**: `suggested_area: unsorted`로 두고 curate 때 사용자에게 텔레그램 질문 (주 1회 묶어서)
- **복습 피로**: 일일 상한 5개 고정, 밀리면 오래된 것 우선
- **볼트 안전**: 스킬은 `00_Inbox`, `02_Areas`, `03_Resources`, `04_Archive`, `_System` 외 폴더는 건드리지 않음. 삭제는 절대 안 하고 Archive 이동만

## 8. 구현 로드맵

| Phase | 내용 | 완료 기준 |
|---|---|---|
| **1** | 볼트 구조 + 템플릿 3종 + `learn-ingest` (수동 실행) + 동영상 전사 파이프라인(yt-dlp/Whisper) | 유튜브·인스타 링크 → 스크립트 전문 포함 Inbox 노트 자동 생성 |
| **2** | 텔레그램 수신 연동 (봇 → Queue → ingest) | 폰에서 링크 전송 → 1분 내 Inbox 노트 + 확인 답장 |
| **3** | `learn-curate` (승격/병합/MOC) + `learn-garden` (링크·태그 정비) | 1주 치 Inbox가 주제 노트로 정리되고, 노트 간 링크·태그가 자동 갱신됨 |
| **4** | `learn-retro` (SRS + 텔레그램 카드 + 응답 처리) | 매일 아침 복습 카드 수신, 버튼 응답이 interval에 반영 |
| **5** | `learn-weekly` + 파라미터 튜닝 | 첫 주간 리트로 리포트 수신 |

Phase 1~2만 돼도 "던지면 정리되는" 핵심 가치가 나오므로, 3~5는 데이터가 쌓인 뒤 붙이는 것을 권장.
